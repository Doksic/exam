# SOLID: Open/Closed Principle (OCP)

Цей проект демонструє розуміння та застосування принципу відкритості/закритості (Open/Closed Principle) згідно з принципами SOLID мовою TypeScript.

## 📌 Суть принципу (OCP)
Програмні сутності (класи, модулі, функції) повинні бути **відкриті для розширення**, але **закриті для модифікації**. Це означає, що нова функціональність має додаватися шляхом написання нового коду, а не шляхом зміни існуючого (що може призвести до багів у вже працюючому функціоналі).

## 📂 Структура проекту
- `src/OCP/antipattern` — Демонстрація поганої архітектури. Клас `PaymentProcessor` має жорстко зашиті умови (`if/else`) для різних типів платежів. Будь-який новий платіж вимагає модифікації цього класу.
- `src/OCP/refactored` — Демонстрація правильної архітектури. Використовується інтерфейс `PaymentMethod`. Клас `PaymentProcessor` приймає абстракцію. Додавання нового методу оплати (наприклад, `ApplePayPayment`) відбувається шляхом створення нового класу, без зміни коду `PaymentProcessor`.

## 🚀 Як запустити

1. Встановіть залежності:
   \`\`\`bash
   npm install
   \`\`\`

2. Запуск Антипатерну:
   \`\`\`bash
   npm run start:antipattern
   \`\`\`

3. Запуск Відрефактореного коду (Правильна архітектура):
   \`\`\`bash
   npm run start:refactored
   \`\`\`
